# 09 - The North Test

Version: 0.0.1
Last updated: 2026-07-07
Author: Dru + GPT
Reason for change: Defines the decision framework every feature must pass before becoming part of North.

## Purpose

The North Test protects the philosophy of the company.

Every feature, conversation, animation, notification, chart, integration, and design decision must pass this test.

If it fails, it is parked, redesigned, or removed.

## The Ten Questions

1. Does this help the user make a better decision?
2. Does this reduce confusion?
3. Does this build trust?
4. Does this explain why?
5. Does this respect the user's agency?
6. Does this reduce friction?
7. Does this fit naturally into someone's life?
8. Does this avoid shame, guilt and manipulation?
9. Does this build confidence instead of dependency?
10. Would Nova genuinely do this?

## Automatic Failures

A feature fails immediately if it:

- creates fear to drive engagement
- creates guilt to encourage action
- rewards screen time instead of real life
- hides reasoning behind recommendations
- shames the user
- compares users in unhealthy ways
- moves user-controlled layouts without permission
- adds complexity without increasing understanding
- makes North feel less human

## The Final Question

Before building anything, ask:

Does this help someone keep heading North?

If yes, build it.

If no, park it.

## Founder Principle

Features are not built because they are possible.

Features are built because they strengthen the companion.
