# 00 - Product Proposal

Version: 0.0.1
Last updated: 2026-07-07
Author: Bodhi + GPT
Reason for change: Captures the phase plan and product execution proposal after the company foundation phase.

## Purpose

This proposal defines how North moves from company foundation into product design, visual design, prototype, engineering, beta and launch.

North should move deliberately.

No rushing.

No drifting.

No building features just because they are exciting.

## Current Status

Phase 0: Company Foundation is complete enough to move forward.

The Codex now defines:

- why North exists
- who North serves
- how Nova speaks
- how Nova thinks
- how North should feel
- what North will never become
- how features are tested
- how ideas are parked

## Phase 1: Product Design

Goal: define the full product experience before visual design or code.

### Deliverables

- The Travellers
- The North Journey
- Information Architecture
- User Flows
- North Ecosystem
- Experience blueprints
- Screen blueprints

## Phase 2: Visual Design

Goal: make North visually recognisable and emotionally consistent.

### Deliverables

- Visual identity
- Nova Aurora direction
- Light and dark themes
- Typography
- Component system
- Motion system
- Iconography
- Exercise visual language
- Chart language

## Phase 3: Prototype

Goal: create a clickable mobile-first prototype before engineering.

### Deliverables

- First launch prototype
- Today prototype
- Workout logging prototype
- Nova conversation prototype
- Journey prototype
- Milestone / Time Capsule prototype

## Phase 4: Engineering

Goal: build the first working private alpha.

### Deliverables

- Tech stack decision
- App architecture
- Database schema
- Authentication
- User profiles
- Workout logging
- Exercise library
- Basic progress tracking
- Nova integration
- Hosting setup

## Phase 5: Private Beta

Goal: test North with a small group of real travellers.

### Deliverables

- Beta onboarding
- Feedback system
- Bug tracking
- User interviews
- Iteration plan

## Phase 6: Launch

Goal: release North publicly only when the core experience is genuinely loved.

### Deliverables

- Website
- App Store / Play Store path
- Social channels
- Launch messaging
- Pricing model
- Support system

## Working Principle

Never build something because it is exciting.

Build it because the foundation beneath it is ready.

## Founder Note

North should be built the same way the original transformation happened:

one layer at a time, one good decision at a time.
